BACnet Raw NPDU

VTS provides the capability to transmit any BACnet NPDU. This includes any network layer message or any application layer messsage. When you use this method of transmitting, you must provide the entire NPDU. When you use the target device dialog window prior to sending, only the MAC-layer destination address and the port from the target device window are used by VTS. The destination and source routing information from the target device dialog is ignored and must be defined in the raw NPDU itself.

You can use this operation to transmit messages that normally cannot be sent using the standard VTS menu selections. Examples include BACnet-ComplexACK-PDUs and the individual segments of segmented message.

Three examples of raw NPDU's are shown:

  1. A network Who-Is-Router-to-Network Message.
  2. An application Who-Is-Request with destination routing.
  3. An application Who-Is-Request with no destination routing.

The following is a step-by-step procedure for transmitting a Raw NPDU Message.

  1. Type the hex OCTET STRING for the NPDU. This must include all source and destination routing information if required. The NPDU will be transmitted exactly as provided.
  2. Make sure you have selected a target device.
  3. Click the "Send" button.
