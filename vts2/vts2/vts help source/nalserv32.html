NALSERV32 Functions
NALSERV32.DLL provides the BACnet client application and network layer managment for VTS using up to four ports to interface to BACnet devices through the data link/physical layers. The types of ports supported by NALSERV32 are:

  • Up to two Ethernet and/or ARCNET adapters in any combination
  • Up to one BACnet MS/TP port
  • Up to one BACnet PTP port
NALSERV32 can be configured to act as an active or passive BACnet device. As an active BACnet device, NALSERV32 acts on VTS's behalf to:
  • Respond to ReadProperty-Requests for VTS's device object
  • Respond to Who-Is-Requests with I-Am-Requests
  • Initiate periodic Who-Is-Requests and I-Am-Requests
  • Initiate periodic TimeSynchronization-Requests
NALSERV32 performs all BACnet segmentation on incoming and outgoing packets, so VTS can see and provide NPDUs atomically to NALSERV32 no matter how large. NALSERV sends and receives the appropriate segmented requests/responses and segment acks relieving VTS of having to deal with segmentation.

NALSERV32 can be configured to act promiscuously and accept and report all BACnet packets to all BACnet devices to VTS. In the non-promiscuous mode, NALSERV32 accepts only BACnet packets sent to its Ethernet/ARCNET adapter, its MS/TP node and when connected to a BACnet PTP device. NALSERV32 always accpets and reports broadcast BACnet packets.

Setting up NALSERV32
When NALSERV32 is initialized by VTS, its configuration parameters are read from the NALSERV32.INI file which is stored in the \Windows or \WINNT folder. If there is no NALSERV32.INI file, the parameters which would have been defined in it are set to preset default values by NALSERV32.DLL.

NALSERV32's operating parameters, which include the properties for VTS/NALSERV32's device object, can be changed using two methods:

  • Using the NALSERV32 configuration dialog
  • Changing the parameters in NALSERV32.INI

Using NALSERV
When you start VTS, it initializes NALSERV32. NALSERV32 provides its own status dialog and is initially minimized as an icon in the Windows taskbar in the bottom right-hand corner of your display screen.

Clicking on the NALSERV32 icon restores its status dialog as shown below.

The NALSERV32 dialog is used to indicate protocol activity in the form of BACnet packets in hex format. When you close the NALSERV32 window, it returns to its iconic form in the Windows taskbar. The NALSERV32 dialog is closed and disappears from the taskbar when you exit VTS.

The areas of the status dialog are described below:

  • Menus:


  • Status Log indicates critical NALSERV32 events and diagnostics including:
    • startup of NALSERV32 including adapter bindings
    • failed transmissions
    • listener thread termination due to error
    • received NPDUs ignored due to:
      • invalid packet length
      • unsupported/invalid BACnet version
      • no free space available
      • no invoke IDs available
      • unknown invoke ID
  • BACnet Packets shows BACnet received and transmitted packets including MS/TP and PTP frames in hex. You can optionally select that the network layer be interpreted using the View menu.

    The packet displays are described below:

  • Outstanding client/server InvokeIds indicates the number of client (cid) and server (sid) Invoke IDs that are being processed by NALSERV32.
  • MAC Stats indicates MAC level and error statistics.
    • RX indicates the number of packets received
    • TX indicates the number of packets transmitted
    • TO indicates the number of packets transmitted for which there was no response
    • ERROR indicates the number of packets which had errors
    • NOMA indicates the number of packets received for which no MAC-layer buffers were available
    • MAUSED indicates the current number of MAC-layer buffers used
    • MAXUSED indicates the maximum number of MAC-layer buffers used
