VT-Open-Request
The following is a step-by-step procedure for making a standard VT-Open-Request.

  1. Select the 'VT-class' from the drop-down list.
  2. Type the decimal value for the 'Local VT Session Identifier' (0-255).
  3. Make sure you have selected a target device.
  4. Click the "VT Open" button. VTS automatically displays the VT-Data/VT-Close dialog.

VT-Data-Request
The following is a step-by-step procedure for making a standard VT-Data-Request.

You cannot reach this point unless you have first opened a VT session as described in steps 1-4.

  1. Type the decimal value for the remote 'VT-session Identifier' (0-255).
  2. Type the decimal value for the 'VT-data Flag' (0-1). Note that each time you click the "VT Data" button, VTS automatically flips the value of the 'VT-data Flag'.
  3. Type the OCTET STRING as a set of characters for 'VT-new Data'.
  4. Click the "VT Data" button. The corresponding encoding in a transmitted packet for this example is shown below in order to highlight the OCTET STRING encoding.

After you click the "VT Data" button, VTS automatically flips the 'VT-data Flag' from 0 to 1 or 1 to 0.

VT-Close-Request
The following is a step-by-step procedure for making a standard VT-Close-Request request.

You cannot reach this point unless you have first opened a VT session as described in steps 1-4.

  1. You can repeat steps 7 and 8 until you want to end the virtual terminal session.
  2. Type a 'Remote VT Session Identifier' for the 'List of Remote VT Session Identifiers'.
  3. Click the "VT Close" button.
