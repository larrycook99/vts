General Information
The Visual Test Shell (VTS) is a Windows®-based test suite for the ANSI/ASHRAE Standard 135-1995 BACnet™ protocol. VTS consists of a collection of modules that act primarily as a BACnet client, capable of communicating with BACnet devices though the following data link/physical layers:

  • Ethernet
  • ARCNET
  • BACnet Master-Slave/Token Passing (MS/TP) on EIA-485
  • BACnet Point-to-Point (PTP) on EIA-232

VTS can operate with up to two Network Interface Cards of Ethernet and/or ARCNET type and one each MS/TP and PTP connection using the standard PC serial ports. VTS is compatible with Windows 95/98 and Windows NT.

The software components of VTS are illustrated in the following diagram:

  • VTS.EXE provides the Windows menus and dialogs to perform on-demand testing, interpretation and evaluation of the BACnet protocol with BACnet devices connected through any of the BACnet ports of the VTS PC.
  • BACPRIM32.DLL provides low-level BACnet data conversion and protocol primatives.
  • DUDAPI32.DLL provides an automated test suite.
  • VTSAPI32.DLL provides interface routines for accessing PICs data for automated testing.
  • NALSERV32.DLL provides BACnet network and application-level protocol functions.
  • PTP.DLL provides BACnet PTP protocol functions.
  • MSTP.DLL provides BACnet MS/TP protocol functions.
  • BACMAC95.VXD/BACMACNT.SYS provide the BACnet data link/physical layer functionality for Ethernet and ARCNET.

Operating Modes
VTS can be configured to operate in the following modes:

  • An active or passive BACnet participant
  • A promiscuous or non-promiscuous device

As an active BACnet participant, VTS responds automatically to the following BACnet packets:

  • ReadProperty-Requests to its own Device Instance
  • Broadcast Who-Is-Requests
  • Who-Is-Requests directed to its own Device Instance
  • BACnet segmented packets directed to its own Device Instance (with BACnet-SegmentACK-PDUs)
In addition, the NALSERV32 component of VTS can be configured to periodically transmit broadcast Who-Is-Requests.

VTS can be set up to be totally promiscuous in order to record all BACnet packets to any destination node. In non-promiscuous mode, VTS records only BACnet broadcast packets and packets directed at its MAC-layer address(es).

VTS Desktop
The VTS visual desktop consists of the following primary status windows:

  • VTS Status Window
  • NALSERV32 Status Window
  • PTP Status Window
  • MSTP Status Window

VTS Status Window
When you start VTS using My Computer, Windows Explorer, the Windows Start/Programs Menu or a Windows desktop shortcut, the main VTS dialog window is displayed as shown below.

This window remains visible until you exit VTS using the VTS File/Close menu or the standard close box in the window. You initiate all VTS functions using the VTS menu after which additional VTS windows appear as required by the function.

NALSERV32 Status Window
NALSERV32 has its own status window which is normally minimized in the form of an icon in the Windows taskbar in the bottom righthand corner of your desktop.

Clicking on the NALSERV32 icon restores its status window as shown below.

The NALSERV32 window is used to indicate protocol activity in the form of raw BACnet packets and also to configure BACnet Device properties for VTS. When you close the NALSERV32 window, it returns to its iconic form in the Windows taskbar. The NALSERV32 window is closed and disappears from the taskbar only when you exit VTS. Click here for more details about the NALSERV32 status window.

PTP Status Window
PTP has its own status window which is normally minimized in the form of an icon in the Windows taskbar in the bottom righthand corner of your desktop. This icon appears only if you have configured NALSERV32 for PTP operation.

Clicking on the PTP icon restores its status window as shown below.

The PTP window is used to indicate protocol activity in the form of raw PTP hex frames. When you close the PTP window, it returns to its iconic form in the Windows taskbar. The PTP window is closed and disappears from the taskbar when you exit VTS. Click here for more details about the PTP status window.

MSTP Status Window
MSTP has its own status window which is normally minimized in the form of an icon in the Windows taskbar in the bottom righthand corner of your desktop. This icon appears only if you have configured NALSERV32 for MS/TP operation.

Clicking on the MSTP icon restores its status window as shown below.

The MSTP window is used to indicate protocol activity in the form of raw MS/TP hex frames. When you close the MSTP window, it returns to its iconic form in the Windows taskbar. The MSTP window is closed and disappears from the taskbar when you exit VTS. Click here for more details about the MSTP status window.
