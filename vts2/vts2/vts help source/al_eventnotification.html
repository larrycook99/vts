Selecting a Target Device

Unconfirmed/ConfirmedEventNotification-Request
The following is a step-by-step procedure for making a standard Unconfirmed/ConfirmedEventNotification-Request.

  1. If you want to send a confirmed request, check the "Confirmed?" checkbox. If you want to send an unconfirmed request, do not check the "Confirmed?" checkbox.
  2. Select the 'Notify Type' from the drop-down list.

If you select either an ALARM or EVENT BACnetNotifyType, the resulting dialog is shown below.

If you select an ACK_NOTIFICATION BACnetNotifyType, the resulting dialog is shown below.

The dialogs that are shown below are examples of alarm BACnetNotifyTypes. The dialogs are identical to event BACnetNotifyTypes and are supersets for the parameters of the ACK_NOTIFICATION BACnetNotifyTypes. The point at which the ACK_NOTIFICATION deviates from ALARMS and EVENTS is identified at the appropriate point in the description.

  1. Type the decimal value of the 'Process Identifier'.
  2. Type the decimal value of the 'Priority'.
  3. Type the decimal value of the Device Instance for the 'Initiatiating Device Identifier'.
  4. Select a standard BACnet Object_Type for the 'Event Object Identifier' from the drop-down list.
  5. Enter a decimal Instance for the Object_Type. VTS automatically calculates the 'Event Object Identifier'.

If the 'Time Stamp' is a dateTime choice for the BACnetTimeStamp:

  1. Select the "time/date" radio button.
  2. Select the day, month and weekday from the drop-down lists. Type the year and time. As you type, invalid entries are shown in red, while valid entries are shown in green.

If the 'Time Stamp' is a time choice for the BACnetTimeStamp:

  1. Select the "time" radio button.
  2. Type the time. As you type, invalid entries are shown in red, while valid entries are shown in green.

If the 'Time Stamp' is a sequenceNumber choice for the BACnetTimeStamp:

  1. Select the "sequence" radio button.
  2. Type the sequenceNumber. As you type, invalid entries are shown in red, while valid entries are shown in green.

  1. Type the decimal value of the 'NotificationClass'.
  2. If you want to include an optional 'Message Text', type the text into the 'Message Text' edit box.
  3. Select the 'Event Type' from the drop-down list.

If you are sending an ACK_NOTIFICATION request, no further parameters are required. The dialog will look similar to the one displayed below. You can go to step 20 to complete the request.

If you are sending an ALARM or EVENT request, additional parameters are required.

  1. If you want acknowledgement of the alarm/event check the "ACK required?" checkbox. If you do not want acknowledgement, do not check the "ACK required?" checkbox.
  2. Select the BACnetEventState for the 'From State' from the drop-down list.
  3. Select the BACnetEventState for the 'To State' from the drop-down list.
  4. Check the appropriate Status_Flags to be included with the BACnetNotificationParameters.

If the 'Event Type' you selected is a CHANGE_OF_BITSTRING BACnetEventType:

  1. Type the referenced-bitstring for the BACnetNotificationParameters in binary with the most-significant digit first and the number of digits typed equal to the number of bits used for the primative value. This means leading zeroes are required. The corresponding encoding in a transmitted packet for this example is shown below for reference.
Go to step 20.

If the 'Event Type' you selected is a CHANGE_OF_STATE BACnetEventType:

  1. Select the BACnetPropertyState from the drop-down list. The corresponding New_States for the selected BACnetPropertyState will be displayed in its drop-down list.
  2. Select the New_State from the drop-down list.
Go to step 20.

If the 'Event Type' you selected is a CHANGE_OF_VALUE BACnetEventType and the New_Value is a changed-bit type:

  1. Select the "bits" radio button.
  2. Type the CHANGED_BITS for the BACnetNotificationParameters in binary with the most-significant digit first and the number of digits typed equal to the number of bits used for the primative value. This means leading zeroes are required. The corresponding encoding in a transmitted packet for this example is shown below for reference.
Go to step 20.

If the 'Event Type' you selected is a CHANGE_OF_VALUE BACnetEventType and the New_Value is a CHANGED_VALUE type:

  1. Select the "value" radio button.
  2. Type the CHANGED_VALUE for the BACnetNotificationParameters in REAL number format.
Go to step 20.

If the 'Event Type' you selected is a COMMAND_FAILURE BACnetEventType:

  1. Type the Command_Value for the BACnetNotificationParameters.
  2. Type the Feedback_Value for the BACnetNotificationParameters.
Go to step 20.

If the 'Event Type' you selected is a FLOATING_LIMIT BACnetEventType:

  1. Type the Referenced_Value for the BACnetNotificationParameters in REAL number format.
  2. Type the Setpoint_Value for the BACnetNotificationParameters in REAL number format.
  3. Type the Error_Limit for the BACnetNotificationParameters in REAL number format.
Go to step 20.

If the 'Event Type' you selected is an OUT_OF_RANGE BACnetEventType:

  1. Type the Exceeding_Limit for the BACnetNotificationParameters in REAL number format.
  2. Type the Deadband for the BACnetNotificationParameters in REAL number format.
  3. Type the Exceeded_Limit for the BACnetNotificationParameters in REAL number format.
Go to step 20.

  1. Make sure you have selected a target device.
  2. Click the "Send" button.
